# Screenshot Framing

Use this when the user provides app, website, document, code, dashboard, or PPT screenshots.

## Priority

1. Preserve source screenshot content when the user needs fidelity.
2. Put the screenshot into a clean target-ratio canvas.
3. Use the current CyberBin theme as the background or accent.
4. Regenerate or redesign only when the screenshot is too long, too dense, too blurry, or impossible to read.

## Questions To Resolve

- Should the screenshot be faithful, cleaned up, or redesigned?
- Which slide role will it occupy?
- What ratio is needed: 21:9, 16:10, 4:3, or 1:1?
- Are there sensitive names, accounts, or numbers to hide?

## Do Not

- Do not paste dense screenshot text into a slide as readable body copy.
- Do not stretch screenshots to arbitrary ratios.
- Do not crop away important UI or labels without user intent.
- Do not embed paid template screenshots as assets.
